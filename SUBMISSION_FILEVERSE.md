# Build What Big Tech Won’t (Fileverse Track)

PLOP uses the Fileverse API as its **private orderbook and receipt store**, which is exactly the kind of privacy‑first collaboration layer this track encourages.

## Why it fits the track

- **Encrypted by design:** Orders and receipts are encrypted client‑side and stored as Fileverse ddocs, so sensitive trading intent never touches a centralized database.
- **Programmable + agent‑friendly:** The engine reads and updates ddocs through Fileverse’s simple REST API, enabling automated matching, settlement, and receipt creation.
- **No lock‑in:** All data lives in decentralized, user‑accessible ddocs — users can access their encrypted order history without relying on a proprietary backend.

In short, PLOP uses Fileverse the way Big Tech storage systems can’t: as a **privacy‑native, programmable document layer** for real financial workflows.
